- Man is the "System Programmers Manual"
- Is a very important command. It's the main but not the only source for getting information about Linux command usage
- Is the main source for information about Linux command line usage 
- Use man ato get information about commands, configuration files and more
- Is organized in sections, these are like chapters on a book, with the command man man you can see all the information about the sections in the man command 

Here's an example on how to read a man file: 

NAME OF THE COMMAND
NAME
DESCRIPTION
AUTHOR
REPORTING BUGS
COPYRIGHT
SEE ALSO

**Common elements of man pages**
- summary: shows you how to use a command 
	- items between brackets '[]' are optional
	- if you see **{a | b}** or **a | b** you must choose one
	- And ... means that you may have more of the preceding item
- **man** pages have sections
	- 1 is for end-user commands
	- 8 is for administrator (root) commands
	- 5 is for configuration files
- Many **man** pages have examples near the end only on complex commands
- Use **/sometext** to search for some text 
- And use **q** to get out of the man page
- Use **n** to get to the next page in the man pages
